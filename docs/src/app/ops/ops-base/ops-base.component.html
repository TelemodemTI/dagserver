Below is the detailed documentation of each of the operators implemented within dagserver.

SSH Operator

This operator enables you to execute commands through an SSH connection to a server. It supports private keys and known hosts.

Icon

Param Dialog




Host: hostname of target machine

User: username

Port: portnumber

knowhost file: file that is used to store the public key of the server to which the operator will connect.

Useful Tip:

The ssh-keyscan -t rsa #ip command is used to scan and display the RSA public key of the SSH server at the specified IP address (#ip). This public key can be manually added to the "knownhosts" file on your local system, allowing secure SSH connections without key confirmation prompts in the future. It is useful for automating the setup of SSH connections to known servers.

private key file: ppk file for SSH authentication.

Useful Tip:

PPK (PuTTY Private Key) files are utilized in the PuTTY SSH client environment for authentication to SSH servers using private keys instead of passwords.

File Operator

This operator enables you to read from or write to the content of structured flat files (CSV) or unstructured files and interact with the xcom of the DAG.

Icon

Param Dialog




Mode: read or write

Filepath: location of file

Row Delimiter: delimiter of row

Useful Tip:

If an empty row delimiter is provided, the file content will be treated as unstructured. This means that the content, whether structured or unstructured, associated with the read/write operation will be passed directly to/from the corresponding XCOM.

First Row Titles: first row is column titles.

Xcom: XCOM that will be used in the case of writing mode, to obtain the data.

JDBC Operator

This operator enables you to execute SQL commands in a JDBC-compatible RDMS.

Icon

Param Dialog




Url: database host

User: database user

Pwd: database pwd

Driver: database driver.

Driver Path: location of driver files

XCOM: XCOM that will be associated with the SQL query to be executed.

Useful Tip:

If you need to interact with data from an XCOM generated during the execution of another node within the DAG, it is advisable to use named parameters in your SQL queries.

Mail SMTP Operator

This operator is used for sending emails using an SMTP server.

Icon

Param Dialog




Host: host SMTP server

Port: port SMTP server

User Smtp: user SMTP server

Pwd Smtp: pwd SMTP server

From Mail: from Mail

To Mail: Target mail

Subject: Subject

XCOM: XCOM with the body if aplicable

Useful Tip:

In the case of this stage, you have the option to directly write the body text of the email in the script tab if you do not wish to use the message from another resulting XCOM.

Java Operator

This operator allows you to execute a class located inside an ordinary java file (Jar File)

Icon

Param Dialog




jarPath: This parameter specifies the path that will be used to create the ClassLoader. Ensure that the JAR file you want to execute is located within this path.

className: This parameter indicates the class to be executed from the JAR file.

Useful Tip:

The operator exposes the following methods:

  • It must implement the Serializable interface.
  • It must implement the Callable interface, where T is the type of object that the callable class should return.
  • The callable class should have access to the xcom variables and the arguments (which contain the two aforementioned variables).
